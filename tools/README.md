# Tools Directory

- `tools/recordings`: Saved interaction recordings used to generate crawl-flow config.
- `tools/scripts`: Utility scripts for manual capture and map-processing workflows.
- `tools/samples`: Sample/manual assets that support troubleshooting and local experimentation.

This folder contains operational tooling only. Product runtime code lives under:
- `crawler/`
- `settings-authoring/`
- `apply-runner/`
